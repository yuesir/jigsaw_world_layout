碎片宽高是怎么确定的

一块碎片的最终宽度，由四个独立部分叠加而成。 其中三个来自切割算法，一个来自固定的 margin 常量。 理解这四层，就理解了碎片的全部几何结构。

▸ 第一步
cutChoices：从碎片数量推算 size

用户选择碎片数量后（如 100 片），程序需要决定把图片切成几行几列，每格多宽。 这一步产生核心参数 size——每个碎片核心区的基准边长。

// 从图片短边出发，逐步减小 size 枚举所有合法切法 m = floor(max(imgW, imgH) / 3) // 起始：图片最长边的 1/3 while (m > 10) { rows = floor(imgH / m) cols = floor(imgW / m) nop = rows × cols // 该 size 能切出的碎片总数 if (nop >= minNop) push({nop, rows, cols, size: m}) // 缩小 size：使行数或列数各增加1 s = floor(imgH / (rows+1)) l = floor(imgW / (cols+1)) m = max(s, l) // 取较大值，确保两个方向都+1 }

▸ 第二步
core.width：余数均摊，差值最多 1px

图片宽度往往不能被 cols 整除。size × cols 通常比图片宽少几个像素， 这个余数需要均摊到各列，保证所有碎片的核心区域拼在一起恰好等于图片宽度，无缝隙。

k = imgW % size // 横向总余数 C = floor(k / cols) // 每列均摊几px（通常=0，因 k < cols） z = k % cols // 还剩几px，逐列 +1 消化 for each column: y = size // 先赋基础值 if (k > 0) { k -= C; y += C } // 加均摊量 if (z > 0) { y++; z-- } // 前 z 列各 +1 core.width = y // ← 该列每块碎片的核心宽度
各列 core.width 分布——差值最多 1px，所有列之和 = 图片宽度
行方向同理：core.height 按 imgH % rows 做相同均摊。 结果是前几行高 size+1，其余行高 size，所有行加总恰好等于图片高度。
▸ 第三步
edges.thickness：像素测量锯齿厚度

碎片的四条边各有一个凸起（tab）或凹口（hole），其伸出/缩进的像素厚度 不是固定值，而是根据 core.width 实时在隐藏 canvas 上绘制并测量得到的。

为什么要实时测量？ 锯齿轮廓是贝塞尔曲线，相同的控制点在不同的 core.width 下， 绘制出来的像素厚度比例略有差异。用像素测量代替公式计算，是唯一精确的方法。

// buildMask 初始化阶段，对每个 core.width 值执行一次 canvas.width = core.width canvas.height = core.width // 1. 绘制一段贝塞尔锯齿曲线 drawBezierEdge(side, tab=false) // 2. 读像素，从边缘向内扫，数出非零像素的行数 imageData = getImageData(0, 0, w, h) g = countFilledPixels(imageData, side) // 3. 存入查表数组（之后直接 O(1) 查询） tabHeights[side][core.width] = g // tab 的厚度 holeHeights[side][core.width] = g // hole 的厚度

▸ 第四步
margin：固定的 7px 安全区

每条边外侧还有 7px 的 margin（源码常量 bounds.margin = 7）。 左右各一个，宽度公式里加 2 × 7 = 14。 这个区域用于存放阴影、斜面高光等渲染效果，防止锯齿被裁切。

y = bounds.margin // y = 7（固定常量） piece.width = O.thickness + j + A.thickness + 2*y piece.height = L.thickness + B + M.thickness + 2*y // O=left j=core.width A=right L=top B=core.height M=bottom
